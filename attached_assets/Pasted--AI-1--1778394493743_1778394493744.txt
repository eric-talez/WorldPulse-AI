가능합니다. 이 서비스는 **“구글어스형 글로벌 이슈 지도 + AI 직업 미래 리포트 + 국가별 포럼”**으로 설계하면 됩니다.

## 1. 서비스 한 줄 정의

**전세계 뉴스·전쟁·질병·정치·문화 이슈를 지도에서 보고, 그 변화가 내 직업과 미래 일자리에 어떤 영향을 주는지 AI가 분석해주는 글로벌 인사이트 플랫폼**

예시 이름:

**WorldPulse AI / EarthIssue / JobGlobe / FutureMap / Global Signal**

---

## 2. 핵심 화면 구조

### A. 글로벌 지도 메인

구글어스/윈디처럼 지도를 중심에 둡니다.

지도 위에 레이어를 켭니다.

| 레이어      | 내용                   |
| -------- | -------------------- |
| 뉴스       | 국가별 주요 이슈            |
| 전쟁/분쟁    | 무력 충돌, 시위, 테러, 긴장 지역 |
| 질병       | 감염병, 보건 이슈, WHO 알림   |
| 정치       | 선거, 정권 변화, 법안, 외교    |
| 경제       | 환율, 실업률, 산업 변화       |
| 문화       | K-콘텐츠, 종교, 사회 트렌드    |
| AI 직업 변화 | 국가별 직업군 변화 위험도       |

전쟁/분쟁 데이터는 ACLED 같은 분쟁 데이터 API를 활용할 수 있고, ACLED는 정치폭력·시위 데이터를 제공하는 near-real-time 소스로 소개됩니다. ([ACLED][1])
질병/감염병은 WHO Disease Outbreak News API를 활용할 수 있습니다. WHO API는 diseaseoutbreaknews 항목을 반환하는 엔드포인트를 제공합니다. ([세계보건기구][2])
글로벌 뉴스·지역 언급은 GDELT가 강력합니다. GDELT는 전세계 뉴스 미디어를 기반으로 사회 이벤트, 반응, 담론, 감정 등을 거의 실시간으로 제공하는 오픈 데이터 그래프입니다. ([GDELT Project][3])

## 3. 사용자 흐름

1. 사용자가 지도를 봄
2. 특정 국가 클릭
3. “현재 이슈 요약” 확인
4. “이 나라의 직업군 변화” 확인
5. 사용자가 자기 직업 입력
6. AI가 분석

   * 이 직업이 사라질 가능성
   * AI로 대체되는 업무
   * 오히려 성장하는 업무
   * 앞으로 필요한 역량
   * 국가별 기회
7. 관련 포럼으로 이동
8. 같은 직업군 사람들과 토론

## 4. 예시 기능

### 국가 클릭 시

**대한민국**

* 오늘의 주요 이슈
* 정치/경제/문화/질병/안보 요약
* AI 영향이 큰 직업군 TOP 10
* 성장 가능성이 높은 직업군 TOP 10
* 사용자 직업 분석하기

### 직업 분석 예시

사용자 입력: **세무사**

AI 결과:

| 항목        | 분석                                 |
| --------- | ---------------------------------- |
| 자동화 가능 업무 | 장부 분류, 신고서 초안, 증빙 정리               |
| 인간이 강한 업무 | 절세 전략, 세무조사 대응, 고객 설득              |
| 미래 변화     | 단순 신고 대행 → AI 세무 컨설턴트형 직업          |
| 필요한 능력    | AI 도구 운용, 데이터 분석, 법률 해석, 고액 자산 컨설팅 |
| 추천 방향     | 세무 자동화 SaaS, 업종별 절세 리포트, AI 세무 상담  |

## 5. MVP는 이렇게 시작하면 좋습니다

처음부터 전세계 모든 기능을 넣으면 너무 큽니다. 1차 버전은 이렇게 가야 합니다.

### 1차 MVP

| 기능     | 구현                        |
| ------ | ------------------------- |
| 세계 지도  | Mapbox 또는 CesiumJS        |
| 국가별 뉴스 | GDELT API                 |
| 전쟁/분쟁  | ACLED API                 |
| 질병     | WHO Disease Outbreak News |
| AI 요약  | OpenAI API                |
| 직업 분석  | 국가 + 직업 입력 기반 리포트         |
| 커뮤니티   | 국가별/직업별 게시판               |
| 로그인    | 이메일/소셜 로그인                |

## 6. 기술 구조

사용자님이 Replit/Spring Boot 경험이 있으니 이 구조가 좋습니다.

### 프론트엔드

* React
* TypeScript
* Tailwind
* Mapbox GL 또는 CesiumJS
* 지도 레이어 UI
* 국가별 사이드 패널

### 백엔드

* Spring Boot
* PostgreSQL
* Redis
* Scheduler
* 뉴스 수집 Worker
* AI 분석 API

### AI 파이프라인

```text
뉴스/API 수집
→ 국가/지역/카테고리 분류
→ 중복 기사 제거
→ 신뢰도 점수 계산
→ AI 요약
→ 이슈 영향도 분석
→ 직업군 영향 분석
→ 지도에 표시
```

## 9. 가장 좋은 포지셔닝

단순 뉴스앱이 아니라 이렇게 잡아야 합니다.

**“세상이 어떻게 변하는지 보여주고, 그 변화가 내 직업과 인생에 어떤 영향을 주는지 알려주는 AI 지도 플랫폼”**

즉,
뉴스 플랫폼이 아니라
**미래 대응 플랫폼**입니다.
